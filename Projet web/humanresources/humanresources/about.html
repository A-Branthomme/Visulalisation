Méthodologie

Introduction

Afin de répondre à notre problématique, nous avons choisi de réaliser deux visualisations. La première, à l'échelle plus macro, est une carte de Paris intramuros dans laquelle nous observons les lignes de métro, les stations de métro ainsi que le prix des appartements à proximité de ces stations, pour les années 2014 à 2019. La deuxième, plus analytique, est un BarChart Race, et nous indique le classement évolutif des stations de métro pour lesquelles l'immobilier est le plus cher, de 2014 à 2019.

La donnée principale étudiée dans cette analyse est le prix moyen des ventes d'appartements parisiens. Pour chaque année, une valeur moyenne en €/m² est associée à chaque station de métro, et correspond à la moyenne des prix d'appartements se situant dans un rayon de 570m autour de cette station (distance moyenne entre 2 stations de métro à Paris).

Sont détaillés par la suite les technologies ont utilisées pour traiter les données, réaliser le site internet et créer les visuels. Une explication détaillée des données et des calculs est également fournie dans cette partie.

Technologies utilisées

Traitement des données

La manipulation des différentes Database, le nettoyage des données, les tests ainsi que les calculs ont été réalisés avec Python 3 et Excel. Le traitement des données à grandes échelle a pu être réalisée grace à la librairie Pandas de Python, qui propose en particulier des structures de données et des opérations de manipulation de tableaux numériques.

Visualisations

La première visualisation a été réalisé avec Leaflet qui est une bibliothèse JavaScript de cartographie en ligne développée par Vladimir Agafonkin de CloudMade et de nombreux contributeurs.

La seconde visualisation a été réalisé avec D3.js qui est également une bibliothèque graphique JavaScript qui permet l'affichage de données numériques sous une forme graphique et dynamique.

Technologies web

Le site internet a été réalisé en CSS et en HTML.

Les données

Datasets

Les données proviennent de 3 bases de données distinctes:

  • La première est la base de données des transactions immobilières réalisées dans Paris de 2014 à 2019. On y retrouve l'ensemble des informations concernant les biens: localisation, prix, taille, date de vente, etc. Ce dataset a été récupéré sur le site internet du cadastre
  • La seconde est la base de données des emplacements de bouches de station de métro. On y retrouve le nom des stations de métro, les différentes entrées/sorties et leur localisation. Ce dataset a été récupéré sur le site internet IleDeFrance-Mobilités
  • La troisième est la base de données des stations de métro et lignes correspondantes. Cette dernière permet de lier les stations de métro aux lignes qui la traverse. Ce dataset a été récupéré sur le site internet de la RATP-Mobilités

Nettoyage des données et choix pour la visualisation

Les données immobilières ont été restreintes aux ventes d'appartements dans Paris intramuros.

Les données de métro ont été filtrées sur les stations dans lesquelles passent des lignes de métro (et non RER, trains, etc.), uniquement dans Paris intramuros.

Des calculs matriciels de distances ont dû être nécessaires afin d'associer les appartements aux différentes stations de métro. Celà a pu être possible en utilisant les valeurs de longitude et de latitude. La règle définie est la suivante: si l'appartement est à moins de 570m de la station de métro (distance moyenne entre 2 stations parisiennes), alors il est pris en compte dans le calcul du prix moyen pour cette station.

Ne possédant que les coordonnées géométriques des bouches de métro (et non des stations), nous avons décider de définir le centre de la station comme étant le barycentre de ses différentes bouches de métro.

Un filtrage global a été réalisé sur les données incomplètes et erronées (notamment sur les valeurs de prix au m²). Aussi, certaines valeurs ponctuelles et abérantes faussant les calculs de moyennes ont été retirées.
